# 授权管理操作手册

## 概述

每个客户购买后，你需要为其**生成一个授权码**，客户在应用中输入授权码完成激活（需联网）。激活后服务器返回 **RSA 签名的授权文件**，保存在客户电脑本地。后续即使断网，应用也能通过本地验签+机器码绑定来验证授权，无需联网。

### 防盗原理

```
A 把安装包给 B → B 输入 A 的授权码 → 服务器发现已绑定 A 的机器 → 拒绝
A 把授权文件拷给 B → B 的机器码与文件中绑定的机器码不匹配 → 拒绝
B 试图篡改授权文件 → RSA 签名验证失败 → 拒绝
```

## 一、首次部署：配置 RSA 密钥

### 步骤 1：启动后端服务

```bash
cd backend
mvn spring-boot:run
```

### 步骤 2：获取公钥

后端首次启动时会自动生成 RSA 密钥对，在日志中输出公钥：

```
============================================
  RSA 密钥对已生成并保存到: ./cert-batch-data
  请将以下公钥复制到 Electron 代码中：
  MIIBIjANBgkq...（一长串 Base64）
============================================
```

也可以通过接口获取：

```
GET http://8.152.161.203:18080/api/license/public-key
```

### 步骤 3：嵌入公钥到客户端

将公钥粘贴到 `frontend/electron/main.js` 中的 `LICENSE_PUBLIC_KEY` 常量：

```js
const LICENSE_PUBLIC_KEY = `MIIBIjANBgkq...你复制的公钥...`
```

然后重新打包客户端。

> **重要**：私钥保存在服务器的 `cert-batch-data/license-private.key` 文件中，切勿泄露！

## 二、生成授权码（卖给客户前操作）

### 请求

```
POST http://8.152.161.203:18080/api/license/generate
Content-Type: application/json
```

### 参数说明

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| customer | String | 是 | 客户标识：姓名、微信昵称、手机号、公司名等 |
| days | Integer | 否 | 有效天数。不传表示永久有效 |

### 示例

**永久授权：**
```json
{ "customer": "张三" }
```

**一年授权：**
```json
{ "customer": "李四公司", "days": 365 }
```

### 返回示例

```json
{
  "code": 200,
  "data": {
    "id": 1,
    "licenseKey": "A3KN-P7HM-X2WR-BF6Q",
    "customer": "张三",
    "status": 0,
    "expireAt": null
  }
}
```

> **把 `licenseKey` 发给客户即可。**

## 三、客户激活流程

1. 客户安装并启动应用
2. 应用显示激活页，提示输入授权码
3. 客户输入授权码 → 应用联网调用服务器激活接口
4. 服务器验证授权码 + 绑定机器码 + RSA签名 → 返回签名授权文件
5. 应用保存授权文件到本地 → 进入主界面
6. 后续启动时**本地验签**，无需联网

---

## 四、查看所有授权

```
GET http://8.152.161.203:18080/api/license/list
```

### 状态说明

| status | 含义 |
|--------|------|
| 0 | 未激活（授权码已生成，客户还没输入） |
| 1 | 已激活（客户已使用，绑定机器） |
| 2 | 已禁用（被你手动禁用） |

---

## 五、禁用授权（客户违规时）

禁用后，客户下次联网时授权会被清除。

```
POST http://8.152.161.203:18080/api/license/disable/{id}
```

---

## 六、解绑机器（客户换电脑时）

解绑后，客户可在新电脑上重新输入同一授权码激活。

```
POST http://8.152.161.203:18080/api/license/unbind/{id}
```

---

## 七、常见场景

| 场景 | 操作 |
|------|------|
| 新客户购买 | 生成授权码 → 发给客户 → 客户输入激活 |
| 客户说"授权码已绑定其他设备" | 解绑 → 让客户重新输入 |
| 客户违规/退款 | 禁用 → 客户联网后自动失效 |
| 客户授权到期 | 生成新授权码发给客户 |
| 客户长期离线使用 | 授权到期前本地验签通过即可，不受影响 |

---

## 八、安全说明

| 防护项 | 机制 |
|--------|------|
| 防止拷贝安装包 | 授权码绑定机器码，B 无法激活 |
| 防止拷贝授权文件 | 机器码不匹配，本地验签失败 |
| 防止篡改授权文件 | RSA-2048 签名，无私钥无法伪造 |
| 防止离线绕过 | 本地验签=公钥验签+机器码+过期时间，无需联网 |
| 远程禁用 | 客户联网时后台检查授权状态，被禁用则清除本地授权 |

## 九、离线验证流程图

```
应用启动
  │
  ├── 读取本地 license.json
  │     │
  │     ├── 无文件 → 跳转激活页
  │     │
  │     └── 有文件 → 本地验证
  │           │
  │           ├── 机器码不匹配 → 跳转激活页
  │           ├── 已过期 → 跳转激活页
  │           ├── RSA签名无效 → 跳转激活页
  │           └── 验证通过 → 启动本地后端 → 进入主界面
  │
  └── 后台联网检查（不阻塞用户）
        │
        ├── 联网成功 + 授权有效 → 无操作
        ├── 联网成功 + 授权被禁 → 清除本地授权 → 跳转激活页
        └── 网络不通 → 离线容忍，正常使用
```

## 十、授权码格式

格式为 `XXXX-XXXX-XXXX-XXXX`，由大写字母和数字随机生成，排除易混淆的 I/O/0/1。
